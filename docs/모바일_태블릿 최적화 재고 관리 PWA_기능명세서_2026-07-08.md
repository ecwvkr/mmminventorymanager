## 1. 프로젝트 개요
---
### 한 줄 정의
모바일/태블릿 최적화 재고 관리 PWA

### 제품 목표
매장 직원의 운영 효율성을 극대화하고, 재고 데이터의 정확성을 확보하며, 직관적인 재고 파악 및 발주 자동화를 통해 매장 운영 부담을 경감시킵니다.

### 배경
기존 매장 재고 관리 방식은 실시간 데이터 부족, 수기 입력 오류, 복잡한 시스템으로 인한 비효율성 문제를 겪고 있습니다. 모바일 및 태블릿 환경에 최적화된 PWA 기반의 직관적인 재고 관리 솔루션으로 이러한 문제점을 해결하고, 매장 직원이 손쉽게 재고를 파악하고 발주를 진행할 수 있도록 지원해야 합니다.

### 사용자 문제
매장 직원은 실시간 재고 현황 파악의 어려움, 수기 입력으로 인한 잦은 오류, 기존 재고 관리 시스템의 복잡성 및 수기로 작성하고 전달하는 발주 과정의 불편함을 겪고 있습니다.

### 해결 방안
모바일/태블릿에 최적화된 PWA를 통해 직관적인 UI를 제공하고, 바코드/QR 스캔 기반의 간편한 재고 변동(입고, 출고, 실사) 처리를 지원하여 입력 오류를 최소화합니다. 실시간 대시보드와 자동 발주 시스템을 통해 재고 부족 품목을 빠르게 파악하고, 거래처별 일괄 발주 및 스마트 입고 매핑으로 운영 효율성을 높입니다.

### 차별점
모바일 및 태블릿에 최적화된 PWA 기반의 직관적인 2분할 UI를 제공하여 사용자 경험을 극대화합니다. 카페/레스토랑 특성을 고려한 유통기한 트래킹 및 다중 태그 시스템, 거래처별 일괄 발주 모아보기, 그리고 발주부터 입고까지 재고 수량을 자동으로 연동하는 스마트 매핑 로직을 통해 경쟁 서비스와 차별화된 가치를 제공합니다.

### 타겟 사용자
주로 카페 또는 레스토랑에서 근무하는 매장 직원(파트타이머 포함)으로, 일상적으로 재고를 확인하고, 부족한 품목을 발주하며, 물품 입고 시 재고를 반영하는 업무를 담당하는 사용자입니다. 스마트폰이나 태블릿 사용에 익숙하며, 복잡한 시스템보다는 직관적이고 빠르게 업무를 처리할 수 있는 도구를 선호합니다.

### 사용자 시나리오
매장 직원이 영업 중 재고 부족 알림을 확인하고, 태블릿을 사용하여 바코드 스캔으로 빠르게 부족 품목을 찾아 발주 리스트에 추가합니다. 거래처별로 그룹화된 발주 내역을 확인하고, 클릭 한 번으로 발주를 보낸 후 상태를 '배송중'으로 변경합니다. 물품이 도착하면 '입고 완료' 버튼을 눌러 자동으로 재고에 반영하고, 대시보드를 통해 실시간으로 업데이트된 재고 현황과 유통기한 임박 품목을 한눈에 확인합니다.

### 핵심 지표
재고 실사 시간 30% 단축, 재고 데이터 입력 오류율 50% 감소, 발주 처리 시간 20% 단축

### 리스크
낮은 사용자 채택률(직원 교육 부족 또는 기존 시스템 의존성), PWA의 모바일 브라우저 호환성 문제, 구글 드라이브 등 외부 서비스 연동 시 데이터 보안 및 안정성 문제, 복잡한 매장 환경에서의 바코드 스캔 정확도 저하 가능성.

### 카테고리
> 생산성 / 업무

### 사용자 역할
> User, Admin

### 디바이스 (기기)
> Mobile, Tablet, Web

## 1. 실시간 대시보드 및 재고 현황
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

메인 대시보드에서 재고 요약, 즐겨찾기 품목, 최소 보유 수량 미달 품목, 전체 재고 자산 가치 그래프, 유통기한 임박 품목을 시각적으로 제공하여 실시간으로 매장 재고 상태를 파악하고 선제적으로 대응할 수 있도록 합니다.

**수용 기준:**
```
1. ○ 메인 대시보드에서 실시간으로 업데이트되는 재고 현황(총 품목 수, 부족 품목 수 등)을 확인할 수 있어야 합니다.
2. ○ 최소 보유 수량 미달 품목과 유통기한 임박 품목이 명확히 표시되고, 목록 형태로 조회할 수 있어야 합니다.
3. ○ 자주 사용하는 즐겨찾기 품목을 대시보드에서 빠르게 접근하고 관리할 수 있어야 합니다.
4. ○ 전체 재고의 자산 가치 추이가 기간별 그래프 형태로 시각화되어 제공되어야 합니다.
```

## 2. 직관적인 재고 변동 관리 (입고/출고/실사)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

모바일/태블릿에 최적화된 2분할 UI와 기기 카메라를 활용한 바코드/QR 스캔 기능을 통해 품목을 빠르게 검색하고, 변동 타입(입고, 출고, 실사)을 명확히 지정하여 장바구니에 담은 후 일괄 반영함으로써 재고 입력 오류를 최소화하고 효율성을 높입니다.

**수용 기준:**
```
1. ○ 기기 카메라를 사용하여 바코드/QR 스캔으로 품목을 즉시 검색하고 상세 정보를 확인할 수 있어야 합니다.
2. ○ 품목 선택 시 '입고(+)', '출고(-)', '실사(Set)' 중 재고 변동 타입을 명확히 선택할 수 있어야 합니다.
3. ○ 선택된 품목과 변동 사항을 임시 '장바구니'에 담아두고 일괄적으로 재고에 반영할 수 있어야 합니다.
4. ○ 재고 반영 후 해당 품목의 재고 수량이 정확하게 업데이트되어야 합니다.
```


### 2.1 바코드/QR 스캔 검색
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Mobile, Tablet

기기 카메라를 활용하여 바코드 또는 QR 코드를 스캔하면 품목을 즉시 검색하고 상세 정보(이름, 현재 재고량, 가격 등)를 화면에 표시하는 기능입니다.


#### 2.1.1 카메라 권한 요청

사용자가 바코드 스캔 기능을 처음 사용할 때 기기 카메라 접근 권한을 요청하고, 권한 거부 시 가이드 메시지 표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Mobile, Tablet


#### 2.1.2 바코드/QR 코드 인식 및 검색

카메라로 캡처한 바코드/QR 코드를 파싱하여 품목 데이터베이스에서 검색하고 결과 반환
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Mobile, Tablet


#### 2.1.3 스캔 결과 상세 정보 표시

스캔 완료 후 품목명, 현재 재고 수량, 가격, 유통기한(해당 시 항목) 등의 상세 정보를 화면에 표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Mobile, Tablet


#### 2.1.4 수동 검색 대체 기능

바코드 스캔이 실패한 경우 품목명 또는 SKU를 입력하여 수동으로 검색할 수 있는 기능
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Mobile, Tablet


### 2.2 재고 변동 타입 선택
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Mobile, Tablet

품목 선택 후 '입고(+)', '출고(-)', '실사(Set)' 중 재고 변동의 유형을 명확히 선택할 수 있는 기능으로, 각 변동 타입에 맞는 동작을 수행합니다.


#### 2.2.1 재고 변동 타입 선택 UI

검색된 품목 아래에 '입고(+)', '출고(-)', '실사(Set)' 세 가지 버튼을 제공하여 사용자가 명확히 선택하도록 구성
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: User | **기기**: Mobile, Tablet


#### 2.2.2 변동 수량 입력

## 3. 자동화된 발주 및 입고 처리


## 5. 재고 이력 조회 및 데이터 백업
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중

모든 재고 변동 이력을 상세히 조회할 수 있는 기능을 제공하고, 정기적으로 구글 드라이브 등 외부 저장소에 CSV 파일 형태로 재고 데이터를 자동 백업하여 데이터의 안정성과 관리 연속성을 보장합니다.

**수용 기준:**
```
1. ○ 모든 품목의 재고 변동 내역(일시, 변동 타입, 변동 수량, 담당자 등)을 상세히 필터링하여 조회할 수 있어야 합니다.
2. ○ 재고 데이터가 정기적으로 (예: 매일, 매주) 사용자가 지정한 구글 드라이브 폴더에 CSV 파일 형태로 자동 업로드되어야 합니다.
3. ○ 데이터 백업 주기 및 저장 위치 등의 백업 설정을 변경하고 관리할 수 있어야 합니다.
```


### 5.1 상세 재고 변동 이력 조회
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: Admin | **기기**: Mobile, Tablet, Web

모든 품목의 재고 변동 내역(일시, 변동 타입, 변동 수량, 담당자)을 필터링하고 정렬하여 조회할 수 있는 기능으로, 투명한 재고 관리와 감시를 지원합니다.


#### 5.1.1 재고 변동 이력 데이터 수집

모든 재고 변동(입고, 출고, 실사)이 발생할 때마다 일시, 변동 타입, 변동 수량, 담당자 정보를 기록
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: Admin | **기기**: Mobile, Tablet, Web


#### 5.1.2 이력 조회 화면 UI

모든 재고 변동 이력을 테이블 형태로 표시하고, 품목명, 변동 타입, 변동 수량, 담당자, 일시 등 컬럼 표시
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: Admin | **기기**: Mobile, Tablet, Web


#### 5.1.3 이력 필터링 기능

품목, 변동 타입, 담당자, 날짜 범위 등으로 재고 변동 이력을 필터링하여 조회
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: Admin | **기기**: Mobile, Tablet, Web


#### 5.1.4 이력 정렬 및 페이지네이션

재고 변동 이력을 일시 또는 다른 필드 기준으로 정렬하고, 페이지 단위로 분할하여 표시
